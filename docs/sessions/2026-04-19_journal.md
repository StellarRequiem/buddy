## Decisions made

- **Forest scan is manual-only** — passive 30s polling was burning RAM on the grader and causing memory pressure. On-demand is the right default; automation available via console API.
- **Docker VM at 2.5GB** — forest containers (swarm, postgres, redis, decepticon) use ~100MB total. 2.5GB gives the Linux VM OS overhead room. Saves ~10GB vs previous config.
- **Kubernetes permanently disabled** — not used by any active project. Was eating ~500MB + slowing Docker Desktop startup from 5min → 5s.
- **restart=no on decepticon/dify/open-webui** — these should only run when explicitly needed. Default should be lean.
- **Docker Desktop settings via backend API** — the settings-store.json file gets overwritten by Docker Desktop on quit. The correct approach is POST to the backend socket API.
- **L0 is done, L1 is next** — all code works. The gap is documentation + one-command install/demo.
